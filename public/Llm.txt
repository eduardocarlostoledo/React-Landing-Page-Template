P: ¿Que servicios incluye el servicio de Pentesting ?
R: Seguridad y Pentesting en Aplicaciones MERN (MongoDB, Express, React, Node.js) Se analiza cada petición entre frontend, backend y base de datos para reducir al máximo los riesgos de seguridad.

P: ¿Qué tipo de Pentesting realizan sobre entornos Node.js y React?
R: Nuestro enfoque es el Pentesting de Aplicación (Capa 7). No nos limitamos a escanear puertos, sino que auditamos la lógica del código y la comunicación entre el cliente y el servidor para encontrar fallas que los scanners automáticos ignoran. Se divide en tres pilares: API, Frontend y Cadena de Suministro.

P: ¿Cómo aseguran la integridad de las APIs (Express + Node)?
R: El backend es el corazón de la seguridad. Nos enfocamos en: Prevención de Inyección NoSQL: Verificamos que las consultas a MongoDB estén saneadas mediante esquemas de validación (Zod/Joi), evitando que atacantes manipulen filtros para extraer información sensible. BOLA (Broken Object Level Authorization): Auditamos que un usuario no pueda acceder o modificar recursos de otro (por ejemplo, ver una "changa" ajena) simplemente cambiando un ID en la URL. Validamos la propiedad del objeto en cada petición. Control de Exposición de Datos: Revisamos que las respuestas JSON no incluyan información interna innecesaria, como hashes de contraseñas, códigos de verificación (SMS) o metadatos del servidor que faciliten un ataque.

P: ¿Cómo manejan la seguridad de las librerías de terceros (npm)?
R: Las aplicaciones Node.js dependen de cientos de paquetes externos. Realizamos un Análisis de Dependencias (SCA):Auditamos el árbol de dependencias para identificar paquetes con vulnerabilidades conocidas (CVEs). Implementamos herramientas de monitoreo continuo para alertar si una librería utilizada queda obsoleta o presenta una brecha de seguridad post-lanzamiento.
